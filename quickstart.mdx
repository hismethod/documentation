# Quickstart in 5 Minutes

Follow this guide to complete your first identity verification in under five minutes.

### Step 1: Get Your API Keys

To use the hConnect API, you need a `Client ID` and a `Client Secret`.

1. Log in to the [**hConnect Developer Dashboard**](https://dashboard.hopae.com).
2. Navigate to the **'Applications'** section and create a new app.
3. From your app's detail page, copy the `Client ID` and `Client Secret`. Keep them secure, as they will be used for Basic Authentication in your API requests.

### Step 2: Create a Verification Session

Now, call the API to create a session to verify a user.

<Tabs>
  <Tab title="MitID (Redirect Flow)">
    Redirect flows send the user to the eID provider's website to authenticate. The `redirectUri` is required for this flow.

    ```bash cURL
    curl --request POST \
      --url https://api-gw.hopae.com/v1/verifications \
      --user "YOUR_CLIENT_ID:YOUR_CLIENT_SECRET" \
      --header "Content-Type: application/json" \
      --data '{
        "eid": "mitid",
        "redirectUri": "https://yourapp.com/auth/callback"
      }'
    ```

    **âœ… Success Response:**
    Your application should now redirect the user's browser to the `flowDetails.authorizationUrl`.

    ```json
    {
      "verificationId": "ver_...",
      "status": "initiated",
      "eid": "mitid",
      "flow": "redirect",
      "flowDetails": {
        "authorizationUrl": "REDIRECT_YOUR_CUSTOMER_TO_HERE"
      },
      "expiresAt": "..."
    }
    ```
  </Tab>
  <Tab title="BankID SE (QR Flow)">
    `YOUR_CLIENT_ID` and `YOUR_CLIENT_SECRET` should be replaced with the values from Step 1.

    ```bash cURL
    curl --request POST \
      --url https://api-gw.hopae.com/v1/verifications \
      --user "YOUR_CLIENT_ID:YOUR_CLIENT_SECRET" \
      --header "Content-Type: application/json" \
      --data '{
        "eid": "bankidse"
      }'
    ```

    **âœ… Success Response:**
    Convert the `flowDetails.qrData` into a QR code and display it to your user.

    ```json
    {
      "verificationId": "ver_Abc123Def456",
      "status": "initiated",
      "eid": "bankidse",
      "flow": "qr",
      "flowDetails": {
        "qrData": "RENDER_THIS_QR_CODE_DATA"
      },
      "expiresAt": "2024-07-30T10:05:00.000Z"
    }
    ```
  </Tab>
  <Tab title="Evrotrust (Push Flow)">
    Evrotrust sends a push notification to the user. `userData` is required to identify the user.

    ```bash cURL
    curl --request POST \
      --url https://api-gw.hopae.com/v1/verifications \
      --user "YOUR_CLIENT_ID:YOUR_CLIENT_SECRET" \
      --header "Content-Type: application/json" \
      --data '{
        "eid": "evrotrust",
        "userData": {
          "email": "user@example.com"
        }
      }'
    ```

    **âœ… Success Response:**
    The `flow` is `push`, indicating that a notification has been sent to the user's device.

    ```json
    {
      "verificationId": "ver_...",
      "status": "initiated",
      "eid": "evrotrust",
      "flow": "push",
      "expiresAt": "..."
    }
    ```
  </Tab>
</Tabs>

### Step 3: Handle the Result

After the user authenticates, hConnect's session status will update to `completed`.

- **For Redirect flows,** the user will be sent back to your `redirectUri` with an `authorization_code` in the query parameters.
- **For QR and Push flows,** you can poll the `GET /v1/verifications/{verificationId}` endpoint until the status is `completed`.

**Checking the Final Status (Polling Example for QR/Push):**

```bash cURL
curl --request GET \
  --url https://api-gw.hopae.com/v1/verifications/ver_Abc123Def456 \
  --user "YOUR_CLIENT_ID:YOUR_CLIENT_SECRET"
```

**âœ… Completed Response:**
The `status` will be `completed`, and the `verifiedAttributes` field will contain the user's information.

```json
{
  "verificationId": "ver_Abc123Def456",
  "status": "completed",
  "eid": "bankidse",
  "authorizationCode": "auth_code_for_token_exchange",
  "verifiedAttributes": {
    "id": "pairwise-id-for-this-user",
    "provider_id": "198001011234",
    "given_name": "John",
    "family_name": "Doe",
    "name": "John Doe",
    "birthdate": "1980-01-01"
  },
  "verifiedAt": "2024-07-30T10:01:30.000Z"
}
```

<Note>
  The `authorizationCode` received (either in the redirect URL or the polling response) is a single-use code. Use it to call the [**Token API**](/api-reference/token/create) to get a final `id_token`.
</Note>
